## 3.2 실행을 중단시키지 않고도 브레이크포인트를 사용하는 방법
실행도중에 어떤 일이 일어났는지 자세하게 기록하는것이다.<br>
조건부 브레이크포인트를 찍으면 쉽게 해결될 일을 굳이 로그 커맨드를 추가하느라 고생하는 사람이 많다.
<br>디버거는 브레이크포인트를 찍은 라인에 닿을때마다 기록한다. 변숫값과 실행 스택 트레이스를 디버거가 기록할 것이다.

- 설정에서 직접 수정이 가능하고, 필요한 데이터를 디버거가 기록하도록 고급 설정을 적용할 수 있다.

> 콘솔 화면에 스택 트레이스가 출력되는 모습은 보통 시각화 형태가 아닌,<br>
텍스트 형태를 하고있다. 콘솔이나 로그파일등 모든 텍스트 포맷으로 아웃풋을 저장할 수 있다는것이 장점이다.
