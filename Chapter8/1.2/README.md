## 호출 그래프를 보고 앱의 코드 설계 파악

- 이번엔 시각화 하는 방법을 배울거다. 
- 호출 그래프는 프로파일러가 수집한 데이터를 표현하는 또 다른 방법으로, 객체와 메서드 호출 사이의 관계에 더 집중한다.

이번 실습도 JProfiler를 사용한다. 항상 CPU의 프로파일러 를 먼저 수행한다. 
<br>한번 발열이 있고 나서 실행이 되지않아, 오늘은 넘어가겠다., 8-2의 코드를 실행시키면 된다.

1. JProfiler도 실행을 스택 형태로 나타낸다.
2. 스택을 펼치면 실행시간, 메서드 호출 횟수등의 필수 정보를 얻을 수 있다.

오른쪽 버튼을 클릭하고 Show Call Graph를 선택하면 수집된 실행 데이터를 호출 그래프 형태로 보기좋게 표시된다.
<br>전체 호출 체인을 뜯어보면 더 자세한 정보를 얻을 수 있다.호출 횟수나 세부정보를 제공하지만 주된 관심사는 객체와 메서드 호출간의 관계다.

호출이 어디서 어디로 되는지 알수있고, 메서드마다 CPU 사용시간과 호출 횟수가 표시된다.
