## 디버거로도 충분하지 않다면
디버거는 코드를 탐색할 때 좋지만, 사용할 수 없을때도 있다.

- 코드의 어느 부분이 아웃풋을 내는지 모르는 채 아웃풋 문제를 조사한다.
- 성능 문제를 조사한다.
- 앱 전체가 실패한 크래시를 조사한다.
- 멀티스레드 구현체를 조사한다.

성능 문제는 대부분 디버거로 조사할 수 없다. **로깅 기법과 프로파일링을 같이 사용할 때 실마리를 잡을 수 있다.**
<br>앱이 완전히 멈추는 크래시의 경우 스레드 덤프를 수집해 분석하는 것이 가장 빠른 길이다.

앱에 문제가 생겨 실행이 중단 될 경우 디버거는 사용할 수 없다.

**멀티스레드 앱 개발**은 많은 개발자가 가장 어렵게 느끼는 영역이다.<br>
엄청나게 복잡한 시나리오에서 동작을 이해하려면 디버깅, 모킹, 스터빙 프로 파일링 등의 다양한 기술을 익혀야한다.
