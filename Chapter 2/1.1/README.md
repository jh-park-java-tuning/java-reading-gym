## 2.1 코드 분석만으로는 불충분할 경우
첫째 코드는 선형적이지 않다.<br>
한 라인 한 라인 읽는 대상이 아니다. 이동해가며 어떻게 처리하는지 로직을 파악해야한다.

일종의 미로찾기 같은것이며, 시와 달리 코드는 항상 모든사람에게 동일한 의미를 지닌다.

```java
예제중에 이런 메서드가 있다.
var digits = new StringDigitExtractor(s).extractDigits();
```

여기서 생성자는 객체만 생성할까 아니면 다른 작업도 같이할까?<br>
뒤에 메서드가 수행하는 진짜 코드는 무엇일까?

- 아무리 작은 코드 조각이라도 둘러봐야할 필요가 항상 있다.
- 코드 분석은 같은 로직을 통과하면서도 여러 경로를 만들어낸다. 
- 전체 프로세스를 간단하게 가져갈지, 복잡해져도 자세히 보고 개별 커맨드를 확실하게 이해할지 선택해야한다.
