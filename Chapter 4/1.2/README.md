## 원격 환경에서 조사하기

로컬에서는 IDE가 알아서 디버거를 부착하지만, 원격 환경의 앱을 디버깅할 때는 직접 앱에 디버거 에이전트를 부착해야한다.

```java
java -jar -agentlib:jdwp=transport=dt_socket,
server=y,suspend=n,address=*:5005 app.jar
```
커맨드라인을 복사해서 넣어주면 된다.

여기서의 설정 내용은 다음과 같다.

- transport=dt_socket
    - 디버거 도구가 디버거 에이전트와 통신하는 방식을 지정한 것이다. dt_socket은 네트워클를 통해 TCP/IP에 접속을 하겠다는 뜻이다.<br>
    디버거와 에이전트는 항상 이 방식으로 통신한다.

- server=y:
    - 앱 실행에 부착한 에이전트를 바로 실행한다는 뜻이다. 에이전트는 디버거도구가 자신에게 접속할 때까지 기다리고 이를 통해 앱 실행을 제어한다.
    <br>server=n으로 하면 에이전트를 시작하지않고 바로 접속한다.

- suspend=n:
    - 디버거 도구가 접속하길 기다리지 않고 시작한다는것이다.y로 설정시에 접속할 때 까지 앱을 실행하지않는다.<br>
    예제에서는 문제가 발생하여서 호출 전에 앱을 시작해야한다. 서버 부팅 프로세스에 문제가 있어, 조사를 한다면 디버거 도구가 접속한 후 해야하니 y로하자

- address=*:5005
    - 5005포트로 실행한다. 네트워크 작업이 필요하다.

### 이후 책에 예제가 담겨있지만, 따로 기술하지않겠습니다. P70 ~ P76

- 요즘은 형상관리툴을 많이 사용해서 배포된 앱을 빌드한 소스 코드의 버전에서 확인할 수 있다.<br>
조사하려는 앱에 컴파일된 소스 코드가 로컬에 갖고 있는 소스 코드와 일치하는지 항상 확인하자.
